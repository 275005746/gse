Goal-Spec-Evidence Engineering

GSE

GSE is a workflow skill for long-running AI coding agent projects. It keeps work anchored to goals, scoped specs, focused execution, evidence gates, and reusable learning.

GitHub repository · Catalog metadata · GateHub

Use it for

Agentic engineering workflows that need continuity across sessions.
Spec-driven development with explicit acceptance and evidence.
Project scaffolds that can adapt to Codex, Claude Code, Hermes, WorkBuddy, and other agent hosts.

Install and validate

git clone https://github.com/275005746/gse.git
cd gse
node scripts/validate-gse.mjs --root . --json

Package, install, signing, URL install, command routing, and release evidence checks are included in the repository.

Command model

  • /gse init creates a project-local .gse/ workspace.
  • /gse continue resumes from the current goal map and evidence state.
  • /gse verify runs the focused validation route for the current slice.
  • /gse public-release prepares public release evidence and owner actions.

Trust boundary

This catalog page proves public discoverability for GSE. It does not by itself prove a third-party marketplace approval or host-native slash command execution. Those claims require separate evidence records.
